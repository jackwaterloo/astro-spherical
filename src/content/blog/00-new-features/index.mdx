## New ContentImage Component

I've added a custom `ContentImage` component (`src/components/ContentImage.astro`) to make it easier to render images in blog posts with consistent styling. The component provides:

- **Rounded corners** - Automatically applies rounded styling to images
- **Captions** - Optional caption support displayed below images
- **Flexible imports** - Works with both imported images and URL strings

### Usage

```jsx
import ContentImage from "@components/ContentImage.astro";
import myPhoto from "./my-photo.jpg";

<ContentImage src={myPhoto} alt="Description of image" caption="Optional caption text" />;
```

For remote images, you'll need to specify width and height:

```jsx
<ContentImage
  src="https://example.com/image.jpg"
  alt="Remote image"
  width={800}
  height={600}
  caption="Image from external source"
/>
```

## Disabled Legal Pages

The Terms and Privacy pages are currently disabled but preserved for future use. They've been disabled by:

- Adding `.disabled` extension to page route: `src/pages/legal/[...slug].astro.disabled`
- Adding `.disabled` extension to content: `privacy.md.disabled` and `terms.md.disabled`
- Commenting out footer links in `src/components/Footer.astro`

**To re-enable:** Remove the `.disabled` extension from all three files and uncomment the legal links section in the footer (see comment in file for instructions).

This approach works for any page/route you want to temporarily disable while keeping the code intact.
